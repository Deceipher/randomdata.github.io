We have a new blog

So recently our server crashed. Well... recently, almost two years ago. It pointed out the fact that we didn's spend that much time on system administration and maintenance. The only backup we had was The Wayback Machine. After multiple meetings and people promising that they would maintain the infrastructure nothing happened. No problem at all, but it illustrated the need for a maintenance free, low cost, version controlled solution to blog post hosting.

Today We've build a new blog using Github Pages. This post is about how this page is set up and how to use the repository... yes it could be on our wiki, but I decided not to.

Project structure

The important part is that blog posts can be in markdown or in HTML. Markdown is perhaps faster and easier, HTML has more eyecandy and integrates better with the rest of the blog.


├── assets
│   └── css, js, fonts, sass
├── data
│   └── historic purposes, licensing, README, .zip
├── images
│   └── speaks for itself right
├── html
│   └── directories with one post each in HTML
├── md
│   └── directories with one post each in markdown
├── index.html
├── content.html
└── README.md

HTML

For the HTML I've build some shortcuts that make writing a post easier. An example for an HTML post is documented in the /html/template directory. The example in that directory can be used as reference as well.

  1. Make a directory in /html/ that has a canonical name for the blogpost
  2. Copy the index.html and content.html from the /html/template directory into your blogpost directory
  3. Edit the content.html with your content
  4. Add your post in the root of blogpage repository see /content.html
  5. Save, git add/commit/push... and done

Markdown

The markdown part is supported by Github pages as Jekyll themes. Which theme is used is documented in the _config.yml in the root of the repository. An example of a markdown formatted page can be viewed in the /md/template/ directory. The example in that directory can be used as reference as well. Documenting in markdown also has the benefit of being rendered by Github as well.

  1. Make a directory in /md/ that has a canonical name for the blogpost in markdown style
  2. Copy the index.md from the /md/template directory into your blogpost directory
  3. Edit the index.md with your content
  4. Add your post in the root of blogpage repository see /content.html
  5. Save, git add/commit/push... and done
